# Training Log — 筋トレ記録 PWA

スプレッドシート形式（行=種目 / 列=日付・場所 / 値=`重量,レップ,セット`）の筋トレ記録を、
**ローカルファーストの PWA** として作り直したもの。ジムでセット間に素早く記録するための最適化がしてあります。

## なぜこれか（スプレッドシートの不満の解決）

| スプレッドシートアプリの不満 | このアプリの解決策 |
|---|---|
| スリープで閉じ、再起動が重い | ホーム画面に追加すると**ネイティブ同様に一瞬で復帰**。再ログイン・再同期なし |
| 通信・ログイン待ち | データは**端末内（IndexedDB）**。完全オフライン動作 |
| セル選択→改行入力が面倒 | 種目タップ→**大きな＋/−ステッパー**で数タップ記録。前回値を自動プリフィル |
| 前回どこまで挙げたか分かりにくい | 種目ごとに**「前回」の数値を常時表示**（漸進性過負荷しやすい） |
| 既存6年分の資産 | 既存のスプレッドシートを **TSV/CSV で取り込み**。**TSV書き出し**で元の形式に戻せる |

ネイティブアプリ（App Store 配布）にしなかったのは、更新のたびの審査・ビルドが重く、
この用途には PWA で十分・最速だからです。

## 使い方（iPhone にインストール）

1. このフォルダを **GitHub Pages** で公開（下記）
2. iPhone の **Safari** で公開 URL（例 `https://<ユーザー名>.github.io/tools/training-log/`）を開く
3. 共有ボタン → **「ホーム画面に追加」**
4. 以降はホーム画面のアイコンから起動。オフラインでも即開く

> ホーム画面アプリはスリープ後もアプリ状態を保持するため、復帰が速く、開き直しのだるさが解消されます。

### GitHub Pages の有効化

リポジトリの **Settings → Pages** で、
- **Source**: Deploy from a branch
- **Branch**: `main`（マージ後）または現在の作業ブランチ / フォルダ `/ (root)`

を選ぶと、`https://<ユーザー名>.github.io/tools/training-log/` で配信されます。
（HTTPS 配信が必須。PWA/Service Worker はローカルファイル `file://` では動きません）

## 画面

- **記録**: 上部で日付・場所を切替。種目一覧に「前回」の数値と今日の記録を表示。タップで記録シート。
- **記録シート**: 「これまでの記録」（直近3回）＋ 重量/レップ/セットのステッパー。
  - 「記録する」で1セット追加。**同じ重量×レップは自動でセット数に合算**（例: 20×10 を2回 →「20×10 ×2」）
  - 「前回と同じ」で前回1セット目の値をワンタップ呼び出し
- **履歴**: 種目ごとの全記録を時系列表示
- **種目**: 部位・種目の追加/編集/削除/並べ替え
- **データ**: バックアップと取り込み（下記）

## データの取り込み（既存6年分）

1. Google スプレッドシートを **ファイル → ダウンロード → タブ区切り値(.tsv)** で書き出し
   （またはセル範囲をコピー）
2. アプリの **データ** タブで、テキスト欄に貼り付け（またはファイル選択）
3. **左端（最古）列の年** を入力 → **プレビュー** で期間・件数を確認 → **取り込む**

### 取り込みが期待する形式

- 1行目: 日付（`7/2(木)` など）
- 2行目: 場所（`AF神宮外苑` など）
- A列: 部位、B列: 種目名（A列が無ければB列を種目名として扱う）
- 各セル: 改行区切りで `重量,レップ,セット`（例 `20,10,2`）

年は列に記録されていないため、**左端列の年**を起点に、月がリセット（前の列より小さくなる）するたびに
翌年へ繰り上げて推定します。列が左→右で時系列に並んでいる前提です。プレビューで期間が正しいか確認してから取り込んでください。

## バックアップ

- **JSONで書き出し**: 全データのバックアップ。**データ** タブの「JSONから復元／取り込み」で復元・マージ（種目は名前、セッションは日付+場所で突き合わせ）
- **TSVで書き出し**: 元のスプレッドシート（種目×日付の wide 形式）に戻して Sheets に貼り戻せる

端末内保存のため、機種変更や誤削除に備えて**定期的に JSON バックアップ**を取ってください。

## 開発 / テスト

```bash
# 実ブラウザ(Chromium)での E2E
node test/e2e.mjs

# アイコンの再生成（依存ゼロ・NodeのzlibでPNG生成）
node icons/gen-icons.js
```

- 依存パッケージ・ビルド不要。`index.html` / `app.js` / `sw.js` の静的ファイルのみ。
- データモデル: `exercises` / `sessions` / `logs`（詳細は `app.js` 冒頭コメント）
